## 二、技术架构

### 2.1 技术栈

| 层 | 选型 | 说明 |
|---|---|---|
| 视图 | Vue 3.4 (CDN) | 无构建，直接 `<script>` 引入 production 版 |
| 路由 | 自实现 Hash 路由 | 微信浏览器对 History API 友好但 Hash 兼容性最好 |
| 状态 | `reactive()` Store | 全局单例，自动持久化到 localStorage |
| 样式 | 纯 CSS（内联 `<style>`） | 不依赖任何 UI 框架 |
| 数据 | Mock + localStorage | 无后端依赖，刷新数据保留 |

### 2.2 单文件结构

```
app.html
├─ <head>
│   ├─ 微信 viewport / X5 meta
│   ├─ <style> ~1100 行
│       ├─ 通用变量与组件（手机壳/状态栏/按钮/头像/卡片）
│       ├─ 22 屏专属样式（login / discover / chat / gift / vip ...）
│       └─ 弹层与 Toast
├─ <body>
│   ├─ <div id="app"></div>
│   └─ <script>
│       ├─ Vue3 CDN
│       └─ ~1200 行
│           ├─ Mock 数据（USERS / GIFTS / FEED / RANK / INVITEES）
│           ├─ 全局 store（reactive + watch + localStorage）
│           ├─ 路由系统（currentRoute / go / back / parseRoute）
│           ├─ UI 状态（giftPanel / poster / firstRecharge / toast）
│           ├─ 业务方法 biz（recharge / sendChatMsg / sendGift / withdraw / openVip / callMinute / finishAuth ...）
│           ├─ 公共组件（StatusBar / NavBar / TabBar）
│           ├─ 21 个页面组件（PageLogin / PageDiscover / ...）
│           ├─ 3 个弹层组件（GiftPanel / PosterModal / FirstRechargeModal）
│           ├─ 根 App + 路由分发
│           └─ 微信 H5 适配代码
```

### 2.3 关键技术决策

- **为什么不用构建工具？** 微信内置浏览器对运行时打包接受度高，单文件可直接通过 OSS/CDN 部署，节省工程链路。
- **为什么用 Hash 路由？** 微信内浏览器 History API 在 iOS 14 以下存在边缘 bug；Hash 路由百分百兼容。
- **为什么 Mock + localStorage？** 原型阶段无后端，但要保留付费/聊天等状态，确保产品 demo 流畅。

## 三、路由设计

| 路径 | 页面 | 说明 |
|---|---|---|
| `/login` | 登录页 | 默认进入；登录后跳 onboard 或 discover |
| `/onboard` | 新人资料填写 | 5 步骤，资料未完整时强制访问 |
| `/discover` | 发现/用户列表 | Tab Bar 主入口 |
| `/feed` | 动态广场 | Tab Bar 主入口 |
| `/messages` | 消息列表 | Tab Bar 主入口 |
| `/me` | 个人中心 | Tab Bar 主入口 |
| `/user/:id` | 个人主页 | 来自发现卡片点击 |
| `/chat/:id` | 聊天对话 | 来自消息或个人主页 |
| `/call/:id` | 视频通话 | 实时扣费场景 |
| `/recharge` | 充值钻石 | 所有付费动作回流 |
| `/wallet` | 钱包 | 收益侧 |
| `/withdraw` | 提现 | 实名 + 限额风控 |
| `/vip` | VIP 开通 | 订阅入口 |
| `/auth` | 实名认证 | 3 步流程 |
| `/invite` | 邀请主页 | 裂变核心 |
| `/invite/rank` | 邀请排行榜 | 强激励 |
| `/invite/stats` | 我的战绩 | 数据反馈 |
| `/landing` | 新人落地页 | 邀请链接落地承接 |
| `/settings` | 设置 | 合规出口 |

**路由守卫逻辑：**

```
未登录 → 强制 /login（landing 例外）
已登录但未完成 onboarding → 强制 /onboard
其他 → 自由跳转
```

**3 个弹层（不占路由）：**

- `uiState.giftPanel` — 礼物面板（从底部滑出）
- `uiState.poster` — 邀请海报
- `uiState.firstRecharge` — 首充礼包（登录 2.5s 后自动触发）

## 四、Store 与持久化

### 4.1 字段清单

```js
{
  loggedIn,       // 是否已登录
  onboarded,      // 是否已完成新人引导
  user: {id,name,av,age,city,sign,vip,vipExpire,followers,fans,friends,likes},
  diamonds,       // 钻石余额（虚拟币）
  cash,           // 可提现现金（元）
  isAuthed,       // 是否实名
  chatFreeRemain, // VIP 当日剩余免费聊天数
  chats: [...],   // 会话列表
  transactions: [...], // 流水
  invitedCount,   // 已邀请人数
  invitedPaid,    // 已充值人数
  invitedReward,  // 累计邀请收益
  myRank,         // 邀请榜排名
  inviteCode,     // 我的邀请码
  settings: {invisible, hideLoc, vipOnly, sound, vibrate},
  firstRechargeShown, // 首充弹窗是否已展示
}
```

### 4.2 持久化机制

- `watch(store, ..., {deep:true})` → 任何写操作自动 `localStorage.setItem('yqyr_store', JSON.stringify(store))`
- 启动时 `localStorage.getItem('yqyr_store')` 还原
- 调试：控制台输入 `yqyr.store` 可直接查看，`localStorage.clear()` 重置

## 五、业务规则一致性

完全按照 `原型说明文档.md` 中的规则实现：

### 5.1 钻石规则
- `¥1 = 10 钻石` 充值（`biz.recharge`）
- 钻石不可提现、不可转赠（仅 store.cash 可提现）

### 5.2 消费规则
| 场景 | 普通 | VIP | 实现位置 |
|---|---|---|---|
| 文字聊天 | 1💎/条 | 100条免费/日 | `biz.sendChatMsg` |
| 视频通话 | 6💎/分 | 5💎/分（8折） | `biz.callMinute` |
| 礼物 | 66~5200💎 | 同价 | `biz.sendGift` |

### 5.3 提现规则（`biz.withdraw`）
- 单笔 ≥ ¥10，否则报错
- 手续费 `Math.max(1, amt*0.006)`
- 未实名拦截 → 自动跳 `/auth`

### 5.4 VIP 价格（`PageVip`）
- 月卡 ¥30 / 季卡 ¥68（限时5折，原价135）/ 年卡 ¥198

### 5.5 邀请奖励
- 注册 +88💎，实名 +100💎，首充 +200💎+30% 现金返佣

## 六、微信 H5 专属适配

### 6.1 Meta 标签

```html
<meta name="viewport" content="...user-scalable=no,viewport-fit=cover" />
<meta name="format-detection" content="telephone=no, email=no, address=no" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="x5-orientation" content="portrait" />
<meta name="full-screen" content="yes" />
```

### 6.2 JS 适配（位于 app.html 底部）

| 处理 | 实现 |
|---|---|
| 检测微信浏览器 | `/micromessenger/.test(ua)` |
| 禁用双指缩放 | `touchstart` 拦截 multi-touch |
| 禁用双击放大 | `touchend` 300ms 间隔判定 |
| 屏蔽长按图片菜单 | `contextmenu` 阻止 |
| iPhone 安全区 | `env(safe-area-inset-bottom)` |

### 6.3 微信 JSSDK 集成桩

```js
window.wxBridge = {
  ready()            // wx.config 后调用
  shareWeChat(opts)  // 自定义分享标题/链接/图
  pay(orderData)     // 调起微信支付
}
```

⚠️ **上线时需补充的真实接入代码：**

1. 后端提供 `/api/wx/sign?url=` 接口签名
2. 前端引入 `https://res.wx.qq.com/open/js/jweixin-1.6.0.js`
3. `wx.config({appId, timestamp, nonceStr, signature, jsApiList:[...]})`
4. 微信支付改为后端下单 → 前端 `WeixinJSBridge.invoke('getBrandWCPayRequest', ...)`

## 七、运行与部署

### 7.1 本地预览

直接双击 `app.html` 即可，桌面端会显示 375×812 手机壳；移动端全屏。

### 7.2 微信预览

1. 把 `app.html` 上传到任意 HTTPS 静态服务（七牛/OSS/Vercel/GitHub Pages）
2. 微信扫码访问 URL
3. 注意：自定义分享需在公众号后台配置 JS 接口安全域名

### 7.3 推荐部署链路

```
开发：本地双击预览
↓
预发：Vercel / Surge / OSS
↓
正式：CDN + 公众号绑定域名 + JSSDK 签名服务
```
